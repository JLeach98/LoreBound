# The LoreBound Codex

## First Edition (Version 1.0)

### Investigate Every Story.

# Letter from the Founder

Every meaningful project begins with a question.

For LoreBound, that question was:

> **What if readers could investigate stories instead of merely reading about them?**

This Codex was intentionally written before the application itself. It serves as the governing document for LoreBound and defines its philosophy, architecture, engineering standards, and long-term vision.

Technology will evolve. The principles contained within this Codex should remain the foundation upon which LoreBound is built.

Welcome to LoreBound.

The investigation begins now.

— Jonathon Ray Leach

# Volume I

## Foundations

> Every investigation begins with principles. Before software can be built, its purpose must be understood.

# Chapter 1 — Vision

> LoreBound should make every user feel like the lead investigator of their own fictional archive.

## Purpose

LoreBound exists to help readers, players, and fans investigate fictional worlds rather than merely catalog them.

## Vision Statement

LoreBound is an investigative platform built around Cases, Dossiers, Bonds, Boards, and Investigative Lenses.

## Prime Directive

Every feature implemented within LoreBound should answer one question:

> Does this strengthen the investigation?

If the answer is no, the feature should be reconsidered.
