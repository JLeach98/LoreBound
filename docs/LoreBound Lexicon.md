# LoreBound Lexicon

## Purpose

The LoreBound Lexicon defines the official language of LoreBound.

Every user-facing word should reinforce the feeling that the user is investigating a fictional universe rather than managing files inside a web application.

LoreBound should always sound like investigative software first and cloud software second.

When choosing between technical terminology and immersive terminology:

- prefer clarity first
- immersion second
- generic SaaS wording last

Avoid unnecessary corporate language.

Avoid generic cloud-storage terminology whenever a LoreBound equivalent exists.

# Button Style Guide

Buttons should be concise.

Prefer verbs.

Avoid unnecessary technical wording.

Examples:

- Commit
- Open
- Connect
- Retrieve
- Create
- Investigate
- Review
- Archive

Avoid generic wording when a LoreBound alternative exists.

# Tone

LoreBound should sound:

- Professional
- Organized
- Intelligent
- Investigative
- Premium

Never:

- Corporate
- Overly playful
- Overly dramatic
- Excessively role-playing

The application should feel believable enough that the user forgets they are interacting with traditional cloud software.

# Future Vocabulary Rules

Whenever new UI text is introduced:

1. Check whether an official LoreBound term already exists.

2. Reuse existing terminology whenever possible.

3. If new terminology is required, it should match the investigative tone.

4. Do not introduce SaaS terminology that conflicts with existing LoreBound language.

5. Prefer consistency over novelty.
